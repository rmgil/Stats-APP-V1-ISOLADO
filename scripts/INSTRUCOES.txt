========================================
COMO FUNCIONA
========================================

1. AMBIENTES SEPARADOS:
   - Produção: Onde os utilizadores usam a app
   - Desenvolvimento: Onde você testa mudanças
   - São COMPLETAMENTE separados

2. PROTEÇÃO DE DADOS:
   - Migrações automáticas no startup
   - Backup antes de mudanças importantes
   - Verificação após deploy

========================================
COMANDOS DISPONÍVEIS
========================================

BACKUP (fazer cópia de segurança):
   python scripts/backup_database.py

RESTAURAR (voltar a um backup):
   python scripts/restore_database.py --list  (ver backups)
   python scripts/restore_database.py backup_20251001_150000.sql

COPIAR PRODUÇÃO → DESENVOLVIMENTO:
   PROD_DATABASE_URL=<url_producao> python scripts/sync_prod_to_dev.py
   (emails são mascarados para privacidade)

VERIFICAR DEPLOY:
   python scripts/verify_deployment.py

========================================
O QUE ACONTECE NO DEPLOY
========================================

1. Código novo é enviado
2. Migrações são aplicadas automaticamente
3. Dados existentes são PRESERVADOS
4. Primary admin garantido ativo
